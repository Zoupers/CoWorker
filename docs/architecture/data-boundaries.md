# 数据与信任边界

中文 · [English](data-boundaries.en.md)

[← 返回架构与核心概念](README.md)

Coworker 是本地运行的自主 Agent，但“本地运行”不等于“数据永不离开设备”。它会按任务
需要调用你配置的模型与工具。本页说明默认边界；具体模型服务和第三方集成仍受各自的
隐私政策、日志策略与部署配置约束。

## 默认保存在本机的数据

下列路径都相对于 Coworker 的工作目录，部分路径可以通过配置修改：

| 路径 | 主要内容 |
|---|---|
| `.env`、`providers.json` | Provider 地址、模型和 API Key 等外部配置 |
| `data/admin_config.json` | 管理员令牌及管理页面保存的配置，可能包含 API Key |
| `data/` | 身份、记忆库、任务、收发件箱、附件、截图、运行状态和日志 |
| `.coworker/` | 技能、记忆宫殿、潜意识模式及其用户修改 |
| 容器 `coworker-workspace` 卷 | Agent 可读写的 Git checkout、本地分支、提交和 `.coworker/` |
| 容器 `coworker-state` 卷 | checkout 中 `data/` 指向的持久化身份、记忆和运行状态 |
| 桌面应用的数据目录 | Desktop 的设置、凭据、桥接状态和日志；具体位置由操作系统决定 |

这些文件可能包含对话、提示词、工具参数与结果、网页内容、文件内容和个人信息。
`.env`、`providers.json` 与管理配置是普通本地文件，Coworker 核心不会替你加密它们；
应依赖操作系统权限、磁盘加密和最小权限账户进行保护。配置导出包会包含运行数据和密钥，
应按凭据文件处理。

默认配置不会把整个 `data/` 或 `.coworker/` 自动同步到项目方服务器，Chroma 的匿名遥测
也已显式关闭。但第三方依赖的下载、模型服务和下述工具仍会产生网络请求。

## 可能离开本机的数据

- 模型调用会发送系统提示词，以及完成当前任务所需的对话、记忆、工具结果和附件内容。
  Coworker 不会无条件上传整个工作目录，但 Agent 读取的文件内容可能随后进入模型上下文。
- `visual_analyze` 会把选中的图片或视频发送给配置的视觉模型服务。
- 搜索工具会发送查询词；浏览器工具会访问目标网站，并受到该网站的日志、Cookie 和会话
  策略约束。
- 企业微信、Desktop 桥接及其他通信或 MCP 集成会向对应服务传输消息、附件和协议元数据。
- 安装依赖、Playwright 浏览器或本地嵌入模型时，会连接软件源、浏览器下载源或模型仓库。
- 容器首次创建工作区时会访问 `COWORKER_REPOSITORY_URL` 指定的 Git 仓库；默认是官方
  CoWorker 仓库。将该变量显式设为空可以改用镜像内源码创建本地 Git 基线。

若数据不能交给某个外部服务，不要为 Coworker 配置该服务，也不要让 Agent 读取相关文件。
使用自托管模型只能改变模型这一条边界，不会自动限制搜索、浏览器或其他集成。

## 执行与网络边界

- Coworker 不是安全沙箱。命令、文件和浏览器工具以启动进程的系统用户权限运行；请使用
  专用低权限账户、容器或虚拟机，并只挂载可丢弃或已备份的目录。
- 网页、消息、附件、技能、记忆和模型输出都应视为不可信输入，可能包含提示注入或恶意内容。
- API 默认绑定 `127.0.0.1`。管理员令牌保护管理 API，但当前 v0.x 并非每个路由都具备完整的
  多租户授权边界。不要直接暴露 8000 端口；远程部署要求 TLS、可信 CORS 来源、强通信令牌
  和额外的网络访问控制。详见 [安全策略](../../SECURITY.zh-CN.md)。

## 查看、备份与清理

先停止 Coworker，避免清理时仍有文件写入。在源码仓库中可以先查看 `data/` 的范围：

```bash
uv run python scripts/cleanup.py status
```

需要重置运行数据时，优先先备份再删除：

```bash
uv run python scripts/cleanup.py backup-delete
```

`cleanup.py` 只处理 `data/` 中的运行时文件，并保留 `data/_backups/`；因此
`backup-delete` 不是彻底擦除。它也不会删除 `.env`、`providers.json`、`.coworker/`、
`credentials/`、桌面应用数据或 Docker 卷。需要完整移除时，应逐项检查并删除这些位置及
`data/_backups/`；删除前确认不再需要恢复。容器部署还应检查绑定目录和命名卷，而不是只删除
容器。`docker compose down -v` 会同时删除 `coworker-workspace`、`coworker-state` 和模型
卷，其中可能包含本地提交、身份和记忆；只有在验证备份后才应执行。
